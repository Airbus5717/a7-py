## Function declarations

```txt
add :: fn(a: i32, b: i32) i32 {
    ret a + b
}
```

Key pieces:

- Name
- Parameter list
- Optional return type
- Function body block

## Parameter immutability model

Function parameters are values unless explicitly reference-based.

```txt
increment :: fn(x: ref i32) {
    x.val += 1
}
```

## Function types

Function values can be used through function type signatures.

```txt
Operation :: type fn(i32, i32) i32
```

## Methods pattern

Method-like behavior is expressed with plain functions and explicit receiver parameters.

```txt
length :: fn(v: Vec2) f32 {
    ret sqrt(v.x * v.x + v.y * v.y)
}
```

## Variadic functions

Variadic parameters support `..type` or untyped variadic forms.

```txt
sum :: fn(values: ..i32) i32 {
    total := 0
    for v in values {
        total += v
    }
    ret total
}
```

## Best practices

- Keep signatures explicit for public APIs.
- Prefer smaller functions to simplify semantic diagnostics.
- Use function types for callback-style APIs.
